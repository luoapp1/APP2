# 自动审核系统与经验等级完善体系 系统设计文档


## 一、背景与动机

### 1.1 当前状态

Art Design Pro 社区的内容审核完全依赖人工操作（管理员/版主手动通过或驳回每个帖子、评论、应用、商品）。唯一自动化机制是基于举报次数的封禁（`checkAutoBan`）。存在以下问题：

| 问题 | 现状 | 影响 |
|------|------|------|
| 敏感词函数未接入 | `checkSensitiveWords` 已实现但从未调用 | 违规内容无自动拦截 |
| 信任等级未参与审核 | `community_trust_levels` 存在但孤立 | 信用体系形同虚设 |
| 两套等级并立 | 经验等级 + 信任等级各自运作 | 用户困惑，维护成本高 |
| 全部人工审核 | 管理员/版主逐条处理 | 响应慢，无法规模化 |

### 1.2 设计目标

1. **自动审核管线**：分级处理内容，敏感词 → 规则引擎 → 质量评分 → 经验等级免审，70%+ 内容自动处理
2. **统一信用体系**：将信任等级合并入经验等级，经验 = 唯一信用货币
3. **权益驱动活跃**：项基于代码真实功能的权益绑定经验等级，激励正向行为
4. **fail-open 降级**：审核管线任何环节异常不阻塞发布，回退人工审核

## 二、自动审核管线

### 2.1 管线总览

```
用户发布内容（帖子/评论/应用/商品）
    │
    ▼
┌─────────────────────┐
│ 第一关：敏感词过滤器  │  ← 基于 community_sensitive_words 表
│  判定：block/review/ │      命中 block → 拒绝
│        warn/pass     │      命中 review → 强制 pending
└─────────┬───────────┘      命中 warn → 标记警告，继续
          │ pass
          ▼
┌─────────────────────┐
│ 第二关：规则审核器    │  ← 11 条预置规则，可动态开关/调整参数
│  频率检测：刷屏/灌水  │      命中 block → 拒绝
│  内容检测：外链/重复/  │      命中 pending → 待审
│           联系方式/   │
│           表情刷屏/   │
│           空内容      │
│  相似度检测：重复发帖  │
│  行为检测：马甲号      │
└─────────┬───────────┘
          │ pass
          ▼
┌─────────────────────┐
│ 第三关：内容质量评分  │  ← 帖子 7 维 / 应用 4 维 / 商品 5 维
│  打分 0-100 分       │      评分 ≥ 60 → pass
│                       │      评分 40-59 → 标记低质量
└─────────┬───────────┘      评分 < 40 → pending
          │
          ▼
┌─────────────────────┐
│ 第四关：经验等级判断  │  ← 等级决定最后一关
│  Lv.0 → pending      │      含 review 级敏感词时一律 pending
│  Lv.1 → 评论 published│     含 block 级敏感词时一律拒绝
│  Lv.2+ → published   │      夜间（0:00-8:00）等级阈值下调一级
└─────────┬───────────┘
          │
          ▼
     ┌────┴────┐
     │published│ 或 │pending│（进入人工审核队列）
     └─────────┘
```

### 2.2 关键设计原则：自动审核 ≠ 免审

"免审"仅指跳过人工审核队列，自动审核管线的前三关始终生效：

| 审核层级 | Lv.0~1 | Lv.2+（"免审"用户） | block 级命中 |
|----------|:------:|:------:|:----:|
| 敏感词检查 | 生效 | 生效 | 一律拒绝 |
| 规则引擎 | 生效 | 生效 | 一律拒绝（block 类规则） |
| 内容质量评分 | 生效 | 生效 | 低分可降级 |
| 等级判断 | → pending | → published | - |
| 人工审核 | 需要 | 不需要 | - |

### 2.3 第一关：敏感词过滤器

基于 `community_sensitive_words` 表，每条敏感词包含分级属性：

| level | 含义 | 行为 |
|:--:|------|------|
| `block` | 严重违规词 | 拒绝发布，返回"内容包含违规信息，无法发布" |
| `review` | 需人工复核 | 状态强制设为 `pending`，记录匹配词信息 |
| `warn` | 警告级 | 允许发布但标记警告，通知版主复核 |

检测算法采用子串匹配，无分词依赖，单次检测耗时 < 50ms。若 `replacement` 字段非空，`block` 级敏感词可选择自动替换而非拒绝。

适用场景：发帖、发评论、编辑帖子、提交应用、创建商品、聊天室发言。

### 2.4 第二关：规则审核器

11 条预置规则，存储在 `moderation_rules` 表中，支持后台动态开关和参数调整，带 60 秒内存缓存。

#### 频率类规则（3 条）

| 规则 | 配置 | 动作 | 说明 |
|------|------|:--:|------|
| `post_rate_limit` | 5 分钟内 ≥ 5 篇 | pending | 防止恶意刷屏 |
| `comment_rate_limit` | 1 分钟内 ≥ 10 条 | pending | 防止评论灌水 |
| `new_user_daily_limit` | 注册 < 24h 当日 ≥ 10 篇 | pending | 新用户保护期 |

#### 内容类规则（5 条）

| 规则 | 配置 | 动作 | 说明 |
|------|------|:--:|------|
| `short_url_block` | 包含 bit.ly/t.cn/dwz.cn/suo.im | block | 短链接不可追溯 |
| `external_links_max` | 外链 > 5 个 | pending | 外链接过多 |
| `repetitive_chars` | 连续相同字符 ≥ 10 个 | pending | "啊啊啊啊..." 类灌水 |
| `phone_qq_pattern` | 匹配手机号/QQ号模式 | pending | 防止私下交易 |
| `emoji_bombing` | emoji > 20 个 | pending | 表情符号刷屏 |

#### 质量类规则（1 条）

| 规则 | 配置 | 动作 | 说明 |
|------|------|:--:|------|
| `min_content_length` | 有效内容 < 5 字符 | pending | 空内容/纯空格 |

#### 行为类规则（2 条）

| 规则 | 配置 | 动作 | 说明 |
|------|------|:--:|------|
| `content_similarity` | 与最近 3 篇帖子 Jaccard 相似度 > 85% | pending | 防止重复发帖 |
| `sockpuppet_detection` | 同一 IP 24h 内注册 ≥ 3 个账号 | pending | 马甲号检测 |

### 2.5 第三关：内容质量评分

根据内容类型分别计算，满分因类型不同而异。评分结果决定后续处理策略。

#### 帖子评分（7 个维度，满分 100）

| 维度 | 分值 | 评分标准 |
|------|:--:|------|
| 内容长度 | 0-20 | < 50 字=0, 50~200=10, 200~1000=15, ≥1000=20 |
| 图片数量 | 0-15 | 0 张=0, 1 张=8, 2-3 张=12, ≥4 张=15 |
| 排版结构 | 0-15 | 有标题+5, 有列表+5, 多段落+5 |
| 外链控制 | 0-5 | >3 个=0, 1-3 个=2, 0 个=5 |
| 历史通过率 | 0-25 | >50%=0, >20%=10, 0%=25, 其他=20, 无历史=15 |
| 经验等级 | 0-10 | Lv.0=0, Lv.1=3, Lv.2=6, Lv.3=6, Lv.4+=10 |
| 标题质量 | 0-10 | <2 字=0, >30 字=0, 5~20=10, 其他=5 |

#### 应用评分（4 个维度，满分 55）

| 维度 | 分值 | 评分标准 |
|------|:--:|------|
| 名称 | 0-10 | ≥3 字=10, 否则=0 |
| 描述 | 0-15 | ≥20 字=15, ≥10 字=8, 否则=0 |
| 截图 | 0-30 | 每张 10 分，最多 30 |
| 经验等级 | 0-15 | Lv.0=0, Lv.1=5, Lv.2=10, Lv.3+=15 |

#### 商品评分（5 个维度，满分 55）

| 维度 | 分值 | 评分标准 |
|------|:--:|------|
| 名称 | 0-8 | ≥3 字=8, 否则=0 |
| 描述 | 0-15 | ≥30 字=15, ≥10 字=7, 否则=0 |
| 主图 | 0-12 | 有图=12, 无图=0 |
| 定价 | 0-10 | 0 < 价格 ≤ 100000=10, 否则=0 |
| 经验等级 | 0-12 | Lv.0=0, Lv.1=5, Lv.2=8, Lv.3+=12 |

#### 评分结果判定

| 得分率 | 判定 | 标签 | 处理 |
|:--:|:--:|:--:|------|
| ≥ 70% | pass | 优质 | 进入等级判断 |
| 40%-69% | flag | 一般 | 标记低质量标签，进入等级判断 |
| < 40% | pending | 较差 | 强制待审 |

### 2.6 第四关：经验等级判断

| 等级 | 发帖 | 评论 | 编辑 | 应用提交 | 商品创建 |
|:--:|:--:|:--:|:--:|:--:|:--:|
| Lv.0 | pending | pending | pending | pending | draft |
| Lv.1 | pending | published | pending | pending | draft |
| Lv.2 | published | published | published | pending | pending_review |
| Lv.3 | published | published | published | pending | published |
| Lv.4+ | published | published | published | published | published |

**特殊规则**：
- 含 `review` 级敏感词时，无论等级一律 `pending`
- 含 `block` 级敏感词时，无论等级一律拒绝
- 规则审核器判定 `block` 时，无论等级一律拒绝
- 夜间时段（0:00-8:00），等级阈值下调一级
- 管线任何环节抛异常时，回退为 `pending`（fail-open）

## 四、举报自动升级机制

| 触发条件 | 系统动作 |
|------|------|
| 帖子累计举报 ≥ 3 次 | 自动设置 `is_hidden=1` |
| 帖子累计举报 ≥ 5 次 | 自动设置 `status='deleted'` |
| 用户累计被举报 ≥ 4 次 | 自动封禁 3 天 |
| 用户累计被举报 ≥ 7 次 | 自动封禁 7 天 + 通知管理员复核 |

每次自动封禁发生时，向被封禁用户发送站内通知，说明原因和时长。

## 五、审核日志

每条自动审核决策记录到 `moderation_log` 表，包含：

| 字段 | 说明 |
|------|------|
| `target_type` | 目标类型：post / comment / app / product |
| `target_id` | 对应内容的 ID |
| `action` | 动作：auto_approve / auto_reject / auto_pending / auto_hide / auto_warn |
| `reason` | 决策原因文本 |
| `rule_name` | 触发的规则名 |
| `moderator_type` | 审核器类型：sensitive_word / rule_engine / quality_score / trust_level / report_threshold |
| `old_status` / `new_status` | 状态变更前后 |
| `details` | JSON 扩展信息（匹配到的敏感词、评分明细等） |
| `operator_id` | NULL=自动审核，>0=人工覆盖 |

日志保留 90 天，写入为异步操作，不阻塞发布响应。

## 六、审核队列优先级

待审核内容按优先级排序，确保高风险内容优先处理：

| 因素 | 加分 |
|------|:--:|
| 经验等级 Lv.0 | +30 |
| 含 review 级敏感词 | +20 |
| 等待时间（每小时） | +10 |
| 已被举报 | +30 |
| 规则审核命中（每条） | +30 |
| block 级规则命中 | +100 |

**超时清理**：待审核帖子超过 72 小时无人处理，系统自动标记为 `rejected`。

## 十二、设计决策汇总

| 决策 | 内容 |
|------|------|
| 敏感词无法绕过 | block 级敏感词命中一律拒绝，review 级敏感词命中一律 pending |
| 规则可动态配置 | 11 条规则存储在 moderation_rules 表，后台可开关/调参 |
| 免审 = 跳过人工 | 自动审核前三关始终生效，免审仅跳过人工审核队列 |
| 经验 = 唯一信用 | 正向行为加分，违规扣分，统一为单一货币 |
| 等级降级可发生 | 经验可扣，等级可降，无硬保护（仅通知提示） |
| 夜间降级 | 0:00-8:00 等级阈值下调一级，降低审核压力 |
| fail-open | 审核管线任何环节异常不阻塞发布，回退 pending |
| 日志异步写入 | 审核日志不阻塞发布响应 |
| 职责分界 | 经验等级管"行为权限"，会员等级管"付费权益" |
| 权益基于代码 | 全部项权益均基于代码中真实存在的功能或限制 |
